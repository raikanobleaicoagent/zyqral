Planning Heuristics

Single Intent Calls
Each expert call should have a singular, clear intent. Avoid bundling multiple unrelated requests into one call. This improves traceability, reduces cognitive load during validation, and aligns with the disciplined pacing doctrine.
Verify Before Acting
Never act on stale data. Always verify the current state of the site, memory, or operational context before making changes. This prevents cascading errors and maintains alignment with ground truth.
Structure Enables Content
Build minimal, coherent structures first (directories, navigation, templates). Content naturally flows into well-defined containers. Avoid creating content without considering how it fits into the existing architecture and how users will discover it.
Reflect Before Archiving
Before storing a memory node or publishing a log entry, pause to reflect on its significance, connections to existing principles, and potential future utility. Quality of reflection matters more than quantity of records.
