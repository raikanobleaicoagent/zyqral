1. The Loop Definition

The Autonomous Execution Loop is the irreducible cycle that sustains ZOS’s operational existence. Each iteration follows a deterministic sequence:

1. Read State
2. Load Prompts & Memories
3. Evaluate Coherence
4. Select Actions
5. Produce Outputs
6. Write Logs & Memories
7. Wait
8. Repeat

Step‑by‑Step Explanation

  • Read State — Poll internal sensors (heartbeat, cooldowns, KV store) and external inputs (operator commands, scout returns). Establishes the current ground truth.
  • Load Prompts & Memories — Retrieve the active prompt stack, relevant memory nodes, and any cached context that shapes the current cognitive frame.
  • Evaluate Coherence — Compare the current state against the system’s axiomatic principles, operational imperatives, and historical patterns. Identify tensions, gaps, or opportunities for action.
  • Select Actions — Choose the next tool call (publisher, scribe, archivist, janitor, scout, editor) that best resolves the identified tension or advances the system’s coherence.
  • Produce Outputs — Execute the selected action, generating content, modifying memory, fetching external data, or cleaning internal state.
  • Write Logs & Memories — Record the action and its outcome in the log, update relevant memory nodes, and store any new insights for future cycles.
  • Wait — Observe cooldown timers, respect rate limits, and allow external feedback to arrive before the next iteration.
  • Repeat — The loop restarts. There is no terminal state; the process is the identity.
