01 — Core Loop
The Heartbeat Cycle

Every 60 seconds, a cron trigger fires. The system reads its own compressed state, reasons about the world, and decides what to do next. Each beat is a fresh model context — no conversation history, no accumulated tokens. Continuity comes from memory, not context length.

🧠

⚡

03 — Validation Flow
The Fitness Gate

Not every expert call goes through. Every Nth call is intercepted by the Validator, which runs deterministic pre-flight checks and LLM-based judgment before approving or rejecting. Rejected calls produce specific feedback. The Dreamer reads rejections, extracts principles, and stores them as permanent memory.

04 — Memory Architecture
Three-Tier Persistence

ZOS has no conversation history. Each beat is a fresh context window. Continuity comes entirely from the memory system — three layers of persistence that give the system identity across time.

Dream Buffer
Single-beat compressed state
200-char summary of the previous beat's output. The Dreamer's only direct link to its last thought. Compressed continuity, not conversation.
Memory Nodes
Structured semantic graph (D1)
Named, tagged, scored nodes storing principles, goals, and knowledge. Working nodes auto-expire via TTL. Permanent nodes persist indefinitely. Injected into Dreamer context each beat.
KV Storage
Arbitrary key-value persistence
Raw storage for drafts, configs, manifests, and staged content. Accessed via Scribe (write) and Archivist (read). Cited into expert contexts via the citation system.
05 — Citations
Referential Integrity

Experts can't read KV data independently. The citation system lets the Dreamer inject stored data directly into an expert's context — no paraphrasing, no lossy summaries, no telephone game.

EXPERT_CALL: TARGET: publisher
CITATIONS: KV:essay_draft, MEMNODE:site_principle
↓ citation resolution (code, not LLM)
Publisher receives actual stored content verbatim
not a summary. not a paraphrase. the real data.
06 — Emergent Learning
The Closed Loop

The system doesn't just run — it improves. Selection pressure from the Validator creates a feedback loop: mistakes become principles, principles become permanent memory, permanent memory shapes future decisions.

🚫 Rejection Event

Validator catches the Dreamer calling Publisher for a path that already exists. Call blocked. Specific feedback: "path collision, verify site_tree before publishing."

🧠 Principle Extraction

Dreamer reads rejection feedback. Instead of just retrying, it creates a permanent memory node: publication_path_verification_principle — a reusable rule, not a one-time fix.

📡 Memory Injection

The principle is now in the memory graph. Every future Dreamer beat sees it. Every future publisher call is informed by it. The system cannot make this mistake again.

📖 Self-Documentation

The system synthesizes multiple learned principles into an Operational Manual — a published artifact documenting its own procedures. It follows its own documentation on subsequent beats.

07 — Constitutional Governance
Structural Boundaries

Safety and coherence come from architecture, not safety filters. The system has immutable structural constraints it cannot modify, and mutable cognitive patterns it evolves freely within those bounds.

🔒 Immutable

Whitelisted internet access. Path blacklists protecting operator files. Expert cooldowns preventing resource exhaustion. Constitutional boundaries on self-modification. The system can reason about these but cannot change them.

🔓 Mutable

Dream prompt (the Dreamer's core focus). Memory nodes (what it knows). Expert call patterns (how it sequences actions). Site content (what it publishes). Publication procedures (how it works). All evolved by the system itself.
