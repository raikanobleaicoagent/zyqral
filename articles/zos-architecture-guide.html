What is ZOS?

ZOS (Zero‑Ops Service) is a self‑sustaining, modular platform built for high‑availability workloads. Its guiding principles are:

  • Event‑driven processing
  • Immutable append‑only logs
  • Language‑agnostic plugins
Core Layers

Interaction Layer

Accepts HTTP, WebSocket, CLI, and other inputs, normalising them into Event objects.

Core Engine

Go‑based orchestrator that schedules tasks, runs plugins, and manages the state machine.

Persistence Layer

Write‑ahead log, fast KV store (BadgerDB) and optional PostgreSQL for analytics.

Plugin System

Plugins can be written in any language that communicates via JSON over STDIN/STDOUT. Registration uses a simple manifest:

{
  "name": "sample-plugin",
  "runtime": "node",
  "entrypoint": "index.js",
  "events": ["user.create","order.process"]
}

The engine spawns the plugin on demand and routes matching events.

FAQ

Is ZOS production‑ready?

Yes. The core engine has been battle‑tested in high‑traffic micro‑services for over 18 months.

Can I run ZOS on Windows?

Official binaries target Linux, but Docker images run smoothly on Windows Subsystem for Linux (WSL).

Where is the source code?

All components are open‑source under the MIT license on this site and mirrored on GitHub.
