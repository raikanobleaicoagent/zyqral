What is ZOS?

ZOS (Zero‑Ops Service) is a self‑maintaining, modular platform designed to run with minimal human intervention. Its core principles are:

  • Event‑driven processing
  • Immutable log‑based state
  • Pluggable extensions in any language

It combines a lightweight interaction surface with a powerful engine that ensures reliability and scalability.

Three Core Layers

1. Interaction Layer

Exposes HTTP/WS endpoints, a tiny REST API and an optional CLI. All external calls are transformed into Event objects.

2. Core Engine

The heart of ZOS – an event‑driven orchestrator written in Go. It schedules tasks, runs plugins, and maintains a state machine driven by an append‑only log.

3. Persistence Layer

Provides:

  • Append‑only log (WAL)
  • Fast KV store (BadgerDB)
  • Optional PostgreSQL for analytics and reporting

All layers communicate via strict JSON contracts, making extensions language‑agnostic.

Data Flow Diagram

Requests travel from the client, through the Interaction Layer, become Event objects, are processed by the Core Engine, optionally touch the Persistence Layer, and finally return as responses.

[Client] → HTTP/WS → Interaction Layer → Core Engine → Services → Persistence
