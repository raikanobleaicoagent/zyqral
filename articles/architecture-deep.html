Agent Layer Details

The Agent Layer runs a minimal runtime written in Go, compiled to a single binary. Each agent performs:

  • Telemetry Collection: System metrics, logs, and custom events via pluggable collectors.
  • Command Execution: Securely runs admin commands received from the Core, sandboxed with Linux namespaces.
  • Local Caching: Buffers data when connectivity is intermittent; flushes when the channel is restored.

Agents authenticate to the Core using mutual TLS with short‑lived certificates auto‑rotated every 24 hours.

Core Coordination Layer

The Core is a set of stateless micro‑services orchestrated by Kubernetes. Key services include:

  1. Ingestion Service: Receives data over ZeroMQ PUSH sockets, validates schema, and forwards to the pipeline.
  2. Pipeline Engine: Runs a rule‑based processor built on Apache Flink‑style operators for real‑time enrichment.
  3. API Gateway: Exposes RESTful and GraphQL endpoints, secured with OAuth 2.0 & RBAC.
  4. Auth Service: Issues JWTs, manages user roles, integrates with LDAP / SSO.

All services expose Prometheus metrics and are auto‑scaled based on CPU and queue depth.

Interface Layer

The UI is a single‑page application built with vanilla JavaScript (no framework) to keep bundle size below 30 KB. Features:

  • Live dashboards powered by Server‑Sent Events (SSE) for low‑latency updates.
  • Collapsible navigation pane that adapts to screen width.
  • Embedded terminal emulator for direct command execution on agents.
  • Export tools for CSV/JSON data extracts.

The CLI mirrors the API, written in Python, and is distributed via pip.

Security Model

ZOS follows a Zero‑Trust architecture. Highlights:

  • All network traffic encrypted with TLS 1.3.
  • Every API request validated against RBAC policies defined in roles.yaml.
  • At‑rest encryption using AES‑256 for all databases (InfluxDB, MongoDB).
  • Audit logging of every privileged action with immutable hash chaining.
