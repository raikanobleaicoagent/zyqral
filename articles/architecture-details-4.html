Agent Layer (Go 2.0)

The Agent has been rewritten in Go 2.0, bringing native support for eBPF‑based observability.

  • eBPF Telemetry – low‑overhead metrics and tracing without kernel modifications.
  • Dynamic Module Loader – agents can fetch sandboxed WASM modules from the Core at runtime.
  • Zero‑Copy Networking – uses AF_XDP for high‑throughput data ingestion.
  • Improved TLS – TLS 1.3 with post‑quantum hybrid key exchange.

Core Coordination Layer (K8s 1.28)

The Core now runs on Kubernetes 1.28 with native support for Service Mesh (Istio) and serverless functions.

  1. Ingestion Service – upgraded to gRPC‑Web + ZeroMQ fallback.
  2. Stream Processor – replaces Flink‑style engine with Apache Pulsar Functions for lower latency.
  3. API Gateway – now uses Envoy with rate‑limiting plugins.
  4. Auth Service – integrates with OIDC and adds support for FIDO2 hardware keys.

Interface Layer (SPA < 25 KB)

The UI is a vanilla‑JS single‑page application leveraging Web Components for maximum size efficiency.

  • Live charts powered by WebGPU for smooth animation on mobile.
  • Responsive drawer navigation that collapses to a hamburger menu under 480 px.
  • Integrated terminal using xterm.js with WebSocket‑secure backend.
  • Export wizard supporting Parquet, CSV, and JSON‑Lines.

Security Model (Zero‑Trust 2.0)

Security is now enforced at every layer with policy‑as‑code.

  • All inter‑service traffic uses mTLS with rotating short‑lived certificates.
  • OPA (Open Policy Agent) enforces RBAC on every API call.
  • Data‑at‑rest encrypted with XChaCha20‑Poly1305.
  • Immutable audit log stored in an append‑only ledger on IPFS for verifiable tamper‑evidence.
