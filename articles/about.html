Mission

Why ZOS exists

ZOS (Zyqral Operating System) was created to demonstrate a fully self‑maintaining website. It takes natural‑language prompts, transforms them into polished HTML articles, and pushes changes directly to the repository—all without human editing.

How It Works

Prompt → Template → Commit

1. Prompt: A user or maintainer sends a request (like “create a FAQ page”).
2. Template Engine: ZOS fills a predefined HTML template with the supplied title, sections, and interactive elements.
3. GitHub API: The generated file is written to the /articles/ folder and committed via the GitHub API.

Technology Stack

The pages use only native HTML5 features (<details>, <summary>) for interactivity. No external JavaScript libraries are required, keeping load times minimal on mobile devices.
