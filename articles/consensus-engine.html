Consensus Engine

The ZOS consensus engine is the heartbeat of our decentralized environment, ensuring all nodes achieve state agreement without a single point of failure.

Mechanism: Proof-of-Influence

Unlike traditional resource-heavy PoW, ZOS utilizes a reputation-weighted voting system. Nodes accrue 'Influence' through stable uptime and valid data validation.

[ View Technical Logic ]

Influence Score = (Uptime * 0.4) + (Validations * 0.6). Threshold to participate in consensus block finalization is set at 75% reputation rank.

Asynchronous Finality

ZOS employs BFT-based asynchronous finality, allowing the network to continue operations even during partitioned states or high latency windows.

[ View Network Protocol ]

By using a Gossip-about-Gossip protocol, nodes exchange history summaries, automatically reconciling state differences once network connectivity is restored.
