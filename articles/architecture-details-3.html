Agent Layer

The Agent Layer is a lightweight Go binary that runs on every managed host. Its core duties include:

  • Telemetry – collects metrics, logs, and custom events.
  • Command Execution – runs admin commands received from the Core inside secure Linux namespaces.
  • Local Caching – buffers data during network outages and forwards it once connectivity returns.
  • Security – authenticates to the Core via mutual TLS with automatically rotated short‑lived certificates.

Core Coordination Layer

The Core is a suite of stateless micro‑services deployed on Kubernetes. Major services:

  1. Ingestion Service – receives data via ZeroMQ PUSH sockets, validates schema, and forwards to the processing pipeline.
  2. Pipeline Engine – rule‑based real‑time processing built on a Flink‑style operator model.
  3. API Gateway – offers REST & GraphQL endpoints, protected by OAuth 2.0 and RBAC.
  4. Auth Service – issues JWTs, syncs with LDAP/SSO, and manages role definitions.

All services expose Prometheus metrics and autoscale based on CPU usage and queue depth.

Interface Layer

The UI is a pure‑JavaScript single‑page app (no heavy framework) kept under 30 KB. Highlights:

  • Live dashboards via Server‑Sent Events for sub‑second updates.
  • Responsive collapsible navigation that adapts to mobile screens.
  • Embedded terminal emulator for direct agent command execution.
  • Data export tools for CSV and JSON formats.

The accompanying CLI, written in Python, mirrors the API and can be installed with:

pip install zos-cli

Security Model

ZOS follows a strict Zero‑Trust approach:

  • All traffic encrypted with TLS 1.3.
  • RBAC policies defined in roles.yaml are enforced on every request.
  • At‑rest encryption (AES‑256) protects InfluxDB and MongoDB data stores.
  • Immutable audit logs with hash chaining ensure tamper‑evidence.
