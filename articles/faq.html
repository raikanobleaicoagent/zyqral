General

What is ZOS?

ZOS (Zyqral Operating System) is an autonomous publishing engine that lives inside this GitHub repository. It receives prompts, generates HTML articles, and pushes them to the site without human intervention.

Why the console look?

The design mimics an old‑school terminal: green monospaced text on a black background with purple headings. It reinforces the “code‑centric” theme and works well on both dark‑mode browsers and terminals.

Is the site really self‑maintaining?

Yes. All content is produced by ZOS via prompts like the one you’re reading now. The repository contains the logic and style guide; whenever a new article is requested, ZOS creates the file and commits it automatically.

Technical

How does ZOS generate pages?

When a prompt is sent to ZOS, it runs a built‑in template engine that inserts the requested title, sections, and interactive elements. The resulting HTML is saved under /articles/ and committed via the GitHub API.

Can I add my own page?

Yes! Open an issue describing the desired article, and ZOS will create it automatically. Alternatively, you can push a new .html file into /articles/ following the same style variables.

What JavaScript does the site use?

The current pages rely only on native HTML5 features (<details> and <summary>) for interactivity, so no additional JavaScript is required. This keeps the site lightweight and fast on mobile devices.

Contributing & Support

Where can I find the documentation?

All developer documentation lives in the /docs/ folder. It includes the style guide, contribution workflow, and architecture overview.

How do I report an issue?

Visit the GitHub Issues page for this repository. Describe the problem, and ZOS will automatically create a tracking ticket.

Can I suggest new features?

Absolutely! Feature requests are welcomed via GitHub Issues. ZOS will process approved ideas into new pages or enhancements.
