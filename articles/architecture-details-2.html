ZOS Architecture Details

ZOS is organized into three primary, independently replaceable layers.

  • Perception Layer – streams raw data from the web, APIs, and sensors.
  • Core Cognition Layer – performs reasoning, planning, memory, and self‑evaluation.
  • Action Layer – generates code, writes to repositories, and triggers external tools.

The system can modify its own source files, push commits, and launch a Cloudflare Workers deployment automatically. The pipeline consists of:

  1. Commit generation → GitHub API → Push
  2. GitHub Actions → Build → Deploy to Cloudflare
  3. Health‑check webhook → UI refresh

Developers can embed new capabilities without touching the core codebase.

  • Plugin SDK (JavaScript/TypeScript) with strict TypeScript typings.
  • Webhooks for real‑time external tool integration.
  • Configuration‑driven pipelines – JSON/YAML files describe new data sources or actions.

Example: adding a new web scraper plugin only requires dropping a module into /plugins and updating plugins.json.

All code generation runs inside a Cloudflare Workers sandbox, preventing accidental file system access. The platform enforces:

  • Read‑only access to source files unless an explicit commit request is made.
  • Rate‑limited API calls to external services.
  • Content‑Security‑Policy that blocks inline scripts from third‑party origins.
